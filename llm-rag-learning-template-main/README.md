## 工夫したポイント
1. **[目標達成までの進捗バー]**: 目標達成の進捗バーを理解度と同じ割合で上昇すると1回目に理解度100％であった場合に1回で進捗バーが最大になる。そのため、低い理解度だと全然バーは上昇しないし、逆に高い理解度だとしてもバーの上り幅は上昇するが、たとえ100％でも何回かしないと進捗バーが上がらないように設定した。
2. **[目標を達成したときの演出]**: 目標を設定し、進捗バーがマックスに達すると目標設定回りのボックスがブラウザ上で光るようなアニメーションを追加した。目標を達成した際にalertで知らせてユーザにokボタンを押させるのだがalertをsetTimeout で 0.9秒後 に出すことでalert通知の前にcssでのアニメーション演出を可能にした。
3. **[苦手な分野の表示]**: 苦手な分野を表示する際は、関連概念から抽出して表示しているのだが、そのままだと英語で苦手分野が表示される。そのため、ユーザに分かりやすいように英語で書かれた概念を日本語化し、苦手分野として表示した。苦手分野として表示する基準としては理解度の平均が60%未満のものを表示するようにした。また、その際の平均値も表示している。
4. **[質問、理解度、日時、関連する概念の表示]**: 特に、その時した質問、理解度、日時を記録されたときにローカルストレージも使用し、ブラウザ上で表示した。これにより、ユーザがブラウザを離れたり、何日か経ったあとでも、いつ、どんな内容を学習をしていたか、どこまで理解していたのかを瞬時に分かるようにした。
5. **[履歴のリセット、目標と進捗バーのリセット]**:ローカルストレージを使用しているため、ユーザは理解度を記録すると、グラフや質問、理解度、目標、進捗バーなどの情報をリロードしても記録は永久に残る。そのため、ユーザが履歴や目標、進捗バーをいつでも削除できるようなボタン追加した。このボタンをそれぞれ押すと画面上のグラフ、記録、進捗バーなどがリセットされる。履歴だけを消して、目標はそのままにしておきたい場合や逆に目標をやっぱり変えたいけど履歴は残しておきたいというユーザのためにもボタンは一応、２つに分けた。

## 使い方
1. `my-learning-system.html`をブラウザで開く
2. 初期化する
3. 目標を設定したい場合、目標をブラウザ上で入力し。目標設定ボタンを押す（別に目標は設定してもしなくてもどちらでもよい）
4. 質問内容をブラウザ上で入力して、質問するボタンを押す
5. 質問に対する結果が表示される
6. 結果を見て、理解度のスライドバーを0～100％まで自分で設定して記録するのボタンを押す
7. 質問、理解度、日時、関連する概念の記録と理解度の折れ線グラフが表示される。また、もしその理解度の分野が平均が60%未満であった場合、苦手分野としてその分野と理解度の平均値が表示される
8. 目標を設定していた場合、その目標達成までの進捗バーが上昇する
9. 目表達成の進捗バーが最大になり目標を達成したら、アニメーションの演出が発生し、目標が達成したというalertが表示される
10. 履歴や目標、進捗バーをリセットしたりして、何度でも0から学習できる

## 今後の改善予定
- [やりたいこと1]苦手分野の表示のところで苦手分野の表示基準は理解度60％未満であるが、その分野が記録を重ねていくうちに60%を超えたとしても苦手分野として表示したままになっていたので、基準値を超えたら苦手分野から削除し、かつユーザに苦手分野を克服ことを知らせるようなアクションを追加したいと思った。また、その苦手分野とする理解度の基準値もユーザが自分で設定できるようにすればもっと使用しやすいと思った。
- [やりたいこと2]目標を達成したときのアニメーションが目標設定のまわりのボックスが光るようなアニメーションだけであったので、もっと画面全体を使用した派手な演出にしたらユーザも楽しめるのではないかと思った。
